## ✨ 概要

**mymy** は、AI エージェント、プロジェクトファイル、運用データをひとつのローカルインターフェースにまとめます。ターミナル、ブラウザのタブ、プロンプト、散らばったファイルを行き来する代わりに、ひとつのワークスペースで：

- **登録した LLM プロバイダーでネイティブエージェントを作成**
- **Drive の `/drive/agents`、`/drive/projects`、`/drive/shared` で作業を管理**
- **エージェントのプロンプト（`AGENTS.md`、`SOUL.md`）を編集**
- **エージェントとチャット**、ファイル添付、ツール結果確認、ノート/タスク/目標/財務/投資/カレンダー管理
- **ロック & プライバシー** — PIN ベースのアクセス（クラウドアカウント不要、ログインフロー不要）

**一人企業**のために設計されています — あらゆる役割を兼ねる開発者ファウンダー、残りはエージェントに委ねる人。

## 🚀 機能

| | 機能 | 説明 |
|---|---------|-------------|
| 🔐 | **PIN 認証** | HttpOnly cookie によるサーバー側 PIN セッション。デフォルト PIN `mymy`、設定で変更可能。 |
| 🤖 | **ネイティブエージェント管理** | エージェント作成/削除、プロンプト編集、グローバルなアクティブエージェント選択、登録済み LLM プロバイダーでのチャット。 |
| 📁 | **Drive ワークスペース** | `/drive/projects`、`/drive/agents`、`/drive/shared` の閲覧、編集、アップロード、削除、復元、ビューア。 |
| 🧱 | **サンドボックスランナー** | ファイル書き込み、ターミナル、コード実行、長時間プロセスを専用ランナーで実行。 |
| 💬 | **チャット** | Markdown/コード/検索結果レンダリング、Drive ファイル添付、clarify 要求のインライン処理。 |
| 📅 | **カレンダー** | 予定のスケジュールと管理、プロジェクトに連携。 |
| 📝 | **ノート** | Markdown ノート、PostgreSQL フルテキスト検索（FTS）、タグ、ピン留め機能。 |
| ⚙️ | **設定** | PIN、LLM プロバイダー、エージェント権限、拡張、スキル、Git 連携の構成。 |
| 🌐 | **i18n** | 英語、韓国語、中国語、日本語の完全サポート。 |
| 🎨 | **Linear スタイル UI** | Linear にインスピレーションを受けた集中型ダークテーマ — 一日中見ても快適。 |

## 🏗️ アーキテクチャ

| レイヤー | 技術 | 備考 |
|-------|-----------|-------|
| **フロントエンド** | Vite · React 19 · TypeScript | SPA、SSR 不要 |
| **スタイリング** | Tailwind CSS v4 | CSS 変数デザイントークン |
| **状態** | Zustand · TanStack Query · React Router | 認証 + 設定（localStorage）、サーバー状態（React Query） |
| **バックエンド** | Rust · axum | REST API、サーバーセッション認証、ネイティブエージェントランタイム、Drive、preview proxy |
| **サンドボックスランナー** | Rust · axum · bubblewrap · Firecracker | 隔離されたコマンド/プロセス実行、managed process、preview 転送 |
| **データベース** | PostgreSQL 16 + pgvector | フルテキスト + セマンティック検索をひとつの DB で |
| **インフラ** | Docker Compose | アプリ、DB、API データボリューム、Drive、サンドボックスランナーを起動 |

## ⚡ クイックスタート

### 前提条件

- [Docker](https://docs.docker.com/get-docker/) & Docker Compose
- （ローカル開発用）[bun](https://bun.sh/) ≥ 1.3 および [Rust](https://www.rust-lang.org/) ツールチェーン

### Docker で実行

```bash
# 1. クローン
git clone <リポジトリ-URL> mymy && cd mymy

# 2. 環境設定
cp .env.example .env

# 3. 起動
docker compose up -d --build
```

**http://localhost:33696** を開き、PIN 入力：**`mymy`**

### ローカル開発

```bash
# データベース
docker compose up -d db

# フロントエンド
cd web
bun install
bun run dev      # http://localhost:5173 (HMR)

# バックエンド（別ターミナル）
cd api
DATABASE_URL=postgres://mymy:mymy@localhost:33432/mymy cargo run
```

## 🔧 設定

すべての設定はアプリ内の**設定**ページ（`/settings`）にあります：

| セクション | 構成内容 |
|---------|-------------------|
| **一般** | PIN の変更 |
| **モデル / LLM プロバイダー** | OpenAI 互換、Anthropic、Ollama、ローカルプロバイダーの登録 |
| **エージェント** | エージェントごとのアプリデータ権限 |
| **Git 連携** | GitHub、GitLab、Gitea への接続（ホスト、ポート、SSH エイリアス） |
| **情報** | バージョン & ポート情報 |

設定はブラウザの `localStorage` に保存され、Rust バックエンドと同期されます。

### ポート

mymy は一般的なサービスとの競合を避けるため **33xxx** 範囲を使用します：

| サービス | ポート |
|---------|------|
| Web（フロントエンド） | `33696` |
| API（Rust） | `33697` |
| Sandbox runner | `33698` |
| PostgreSQL | `33432` |
