## ✨ 개요

**mymy**는 AI 에이전트와 코드 프로젝트를 하나의 차분한 인터페이스로 모아줍니다. 터미널, 브라우저 탭, 설정 파일을 오가는 대신, 하나의 대시보드에서:

- **에이전트를 한눈에** — 누가 활성인지, 대기 중인지, 무엇을 실행 중인지
- **프로젝트 진입** — GitHub, GitLab, Gitea에 연결된 프로젝트
- **에이전트와 채팅**, 노트 작성, 캘린더 관리 — 한 곳에서
- **잠금 & 프라이버시** — PIN 기반 접근 (클라우드 계정 없음, 로그인 플로우 없음)

**1인 기업**을 위해 설계되었습니다 — 모든 역할을 겸하는 개발자-파운더, 나머지는 에이전트에게 위임하는 사람.

## 🚀 기능

| | 기능 | 설명 |
|---|---------|-------------|
| 🔐 | **PIN 인증** | 로컬 우선 접근. 기본 PIN `mymy`, 설정에서 변경 가능. 계정 없음, 이메일 없음. |
| 🤖 | **에이전트 관리** | Hermes / OpenClaw 에이전트를 실시간 상태, 아바타, 역할과 함께 조회. |
| 📁 | **프로젝트 워크스페이스** | Git 원격과 연결된 프로젝트 — 진입, 정리, 에이전트 할당. |
| 💬 | **채팅** | 에이전트와 채팅, 프로젝트 또는 일반 주제에 연결된 세션 단위로 정리. |
| 📅 | **캘린더** | 일정 예약 및 관리, 프로젝트에 연결. |
| 📝 | **노트** | 마크다운 노트, PostgreSQL 풀텍스트 검색(FTS), 태그, 고정 기능. |
| ⚙️ | **설정** | 에이전트 시스템(다중 인스턴스: 로컬 + 원격) 및 Git 연동 구성. |
| 🌐 | **i18n** | 영어, 한국어, 중국어, 일본어 완전 지원. |
| 🎨 | **Linear 스타일 UI** | Linear에서 영감받은 집중된 다크 테마 — 하루 종일 봐도 편안한. |

## ⚡ 빠른 시작

### 사전 요구사항

- [Docker](https://docs.docker.com/get-docker/) & Docker Compose
- (로컬 개발용) [bun](https://bun.sh/) ≥ 1.3 및 [Rust](https://www.rust-lang.org/) 툴체인

### Docker로 실행

```bash
# 1. 클론
git clone <저장소-URL> mymy && cd mymy

# 2. 환경 설정
cp .env.example .env

# 3. 실행
docker compose up -d --build
```

**http://localhost:33696** 을 열고 PIN 입력: **`mymy`**

### 로컬 개발

```bash
# 프론트엔드
cd web
bun install
bun run dev      # http://localhost:5173 (HMR)

# 백엔드 (별도 터미널)
cd api
cargo run        # http://localhost:33697
```

## 🔧 설정

모든 설정은 앱 내 **설정** 페이지(`/settings`)에 있습니다:

| 섹션 | 구성 내용 |
|---------|-------------------|
| **일반** | PIN 변경 |
| **에이전트 시스템** | Hermes/OpenClaw 인스턴스 추가 (로컬 자동 감지, 원격 수동) |
| **Git 연동** | GitHub, GitLab, Gitea 연결 (호스트, 포트, SSH 별칭) |
| **정보** | 버전 & 포트 정보 |

설정은 브라우저 `localStorage`에 저장되며 Rust 백엔드와 동기화됩니다.

### 포트

mymy는 일반 서비스와의 충돌을 피해 **33xxx** 대역을 사용합니다:

| 서비스 | 포트 |
|---------|------|
| Web (프론트엔드) | `33696` |
| API (Rust) | `33697` |
| PostgreSQL | `33432` |
